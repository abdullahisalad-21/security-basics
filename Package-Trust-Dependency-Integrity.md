# Security Basics — Package Trust & Dependency Integrity

## Overview
Today’s work reinforced security fundamentals by exploring package trust, dependency integrity, and repository security models in Linux and Windows.

---

## Commands / Work Done
- Verified `.deb` sources using official Google URLs.
- Compared trusted repositories vs manual downloads.
- Reviewed dependency chains and potential security risks.

---

## What I Learned
- Installing from trusted repositories reduces security risks.
- Manual `.deb` installation requires verifying source authenticity.
- Dependency resolution prevents broken or insecure installations.
- Outdated packages can introduce vulnerabilities.

---

## Scenarios
- Installing software securely on Linux.
- Understanding risks of outdated or deprecated libraries.
- Ensuring system integrity during manual installations.

## 1. 🌅 Extended Overview
Today I studied how application software policies reduce organizational risk by controlling what software users can install, run, or access. I learned how standardization, whitelisting, and update enforcement minimize attack surfaces.

## 2. 🎯 Objectives

- Learn how to enforce approved software lists.
- Identify risky software categories and browser extensions.
- Connect software governance to enterprise security.

## 3. 🛠️ Tools and Commands Used

- WDAC policy enforcement

- Browser extension management policies

## 4. 📋 Steps Completed

- Studied software whitelisting and standardization.
- Analyzed browser extension risks and permission models.
- Connected application policies to patching and SIEM visibility.

## 5. 🐞 Problems and Fixes
- No technical issues, but clarified the difference between whitelisting by hash vs certificate.

## 6. 📚 What I Learned
- Application governance is essential for reducing shadow IT.
- Standardizing software simplifies patching and vulnerability management.
- Browser extensions require strict review due to high privileges.
- Policies must include user education to prevent unsafe behavior.
